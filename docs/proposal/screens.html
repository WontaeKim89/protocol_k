Protocol K
화면 기획 (최소 기능 테스트)

Phase 1 의 핵심 흐름(사용자 검색·Try-it + 운영자 변환·검수)을 검증하기 위한 최소 화면 4종. test/ 폴더에 격리한 React + Vite SPA 로 구축하며, 정식 frontend (담당자 조현재) 시작 시 통째 제거합니다.

페르소나

이 화면을 누가 쓰는가

Phase 1 화면이 커버하는 3가지 역할. P2 (Connect Guide) 는 후속 단계.

P1 · USER

일반 사용자

카탈로그에서 도구를 검색하고, 상세 페이지에서 Try-it 으로 즉석 호출 검증.

P2 · DEVELOPER

AI 개발자

endpoint 가져가 Claude Desktop · LangGraph 연결. Connect Guide 페이지는 Phase 2+ 로 미룸.

P3 · OPERATOR

운영자

PDF/HWP 명세 업로드 → LLM 추출 결과 검수 → 카나리아 확인 → 게시.

페이지별 컴포넌트

4페이지 상세

/catalog

Catalog

검색 · 필터 · 카드 그리드
  • 상단 nav + 검색바 (자연어 + 키워드)
  • 좌측 필터: 카테고리 칩 · 기관 · Tier
  • 도구 카드 그리드 (이름 · 짧은 설명 · 태그 · 인기도)
  • 정렬: 인기 · 최신 · 이름
  • 상태: 로딩 스켈레톤 · 빈 결과 · 에러
  • 카드 클릭 → /tools/:slug
/tools/:slug

Tool Detail · Try-it

메타 = mock · Try-it = 실제 호출
  • 헤더: 도구명 · 기관 · Tier 배지 · 태그
  • 설명 (한국어 + 영문)
  • 입력 / 응답 스키마 시각화
  • Try-it 패널 — 스키마 기반 폼 자동 생성
  • 실행 버튼 → 실제 Gateway POST → JSON viewer
  • MCP config 스니펫 (Claude Desktop · LangGraph 등)
  • 함정·주의사항 (격자좌표 · LAWD_CD 등)
/admin/upload

Admin · API Registration

P3 운영자 · mock
  • 드래그-드롭 파일 업로드 (PDF / HWP / OAS YAML)
  • 기본 메타 폼: 기관 · 카테고리 · 태그 · 인증 방식
  • "변환 시작" 버튼 → mock 응답 (slug 부여)
  • 완료 시 /admin/editor/:slug 로 이동
  • 업로드 이력 (마지막 5건, mock)
/admin/editor/:slug

Admin · Tool Editor

P3 운영자 · 3-pane · mock
  • 좌: 원본 명세 (PDF preview placeholder)
  • 중: LLM 생성 OAS YAML 편집기 (diff 보기)
  • 우: 카나리아 호출 결과 + 함정 description 보강 필드
  • 액션: 게시 · 재실행(LLM) · 거부
  • 상태 배지: Pending → Reviewing → Canary → Published

FLOW A · 일반 사용자

   │

   │
   ├─→ 지역명 / 측정소 입력
   │
   ▼

   │
   ▼
JSON 응답 확인 + 호출 시간 + 응답 검증
FLOW B · 운영자 
Admin Upload
   │
   ├─→ PDF/HWP 드래그-드롭 + 메타 입력
   │
   ▼
"변환 시작" (mock) → slug 발급
   │
   ▼
Admin Tool Editor (자동 이동)
   │
   ├─→ 좌측: PDF preview
   ├─→ 중앙: LLM OAS YAML 검수 (mock)
   ├─→ 우측: 카나리아 결과 확인
   │
   ▼
"게시" → Catalog 즉시 노출 (Phase 1 에선 mock 데이터 추가만)

기술 스택

React + Vite + TypeScript

항목선택이유
FrameworkReact 19조현재 인계 친화. 모던 표준.
BuildVite 7dev 서버 빠름, HMR, 셋업 5분.
LangTypeScriptbackend OpenAPI 자동 생성 친화.
RouterReact Router v7표준.
스타일Vanilla CSS + 변수index.html 톤 일관. Tailwind 등 의존성 추가 X.
상태useState글로벌 store 불필요 (Phase 1 규모).
패키지pnpm빠름·디스크 효율.
